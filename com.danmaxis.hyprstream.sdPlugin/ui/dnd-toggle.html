Tap to toggle Do-Not-Disturb on your notification daemon.

The plugin auto-detects whether you have mako or dunst running and uses whichever is available. With mako, the plugin toggles the do-not-disturb mode (makoctl mode -t do-not-disturb). With dunst, it toggles the paused state (dunstctl set-paused toggle).

Visual feedback: green bell with the label ALERTS when notifications are flowing; red bell with a slash and DND ON when paused. The state polls at 1 Hz so toggles from your shell, scripts, or other tools are reflected within a second.

Use this before going live, in meetings, or during deep-focus sessions where popup notifications would be disruptive.
Requires: either makoctl (Arch: sudo pacman -S mako) or dunstctl (Arch: sudo pacman -S dunst) on PATH. If neither is present, pressing the key shows an alert.
