Active mode calls hyprctl killactive — graceful close that lets the app prompt for unsaved changes (e.g. browser "really close?" dialog). Same as your Super+Q keybind.

Workspace mode enumerates every window on your current workspace and sends each a graceful close (hyprctl dispatch closewindow address:0x…). Apps still get to prompt about unsaved changes individually. Issued in a single hyprctl --batch for speed.

Tap-twice-to-confirm: the first tap arms the key — a red ring appears around the X and drains as the confirm window elapses. A second tap inside the window fires the close. Wait out the window (default 3s) and it disarms — nothing happens.
⚠ Workspace mode is destructive. The default Auto confirmation enables a tap-twice gesture so an accidental tap can't wipe your workspace. Switch confirmation to "Tap to fire immediately" only if you really want a one-tap close-all.
